Reproducing Evaluation Results

Abstract

Control groups added Latency in NFVs: An update needed?

The Linux kernel is continuously developing, and the development teams continuously add new versions of libraries with additional sets of features. The cgroups feature of the Linux kernel, widely used by lightweight virtualization technologies, such as Docker or LXC, providing resource isolation, was recently updated. Cgroups underwent a significant revamp from version 1 (v1) to version 2 (v2). Researching the performance difference between these two versions regarding network latencies and throughput enables the usage of containers in time-critical applications for Network Function Virtualization (NFV). Previous work did not consider cgroups as a potential source of latency in NFVs in their evaluations. In this paper, we use commodity hard- and software to measure the difference between v1 and v2 in isolation. Our experiments show that the two versions achieve the same degree of isolation. However, the tail latencies of v1 are higher, which can be explained by a more efficient, \SI{2.4}{\percent} less instruction-consuming implementation of v2. Based on our findings, we recommend using v2 for low-latency, lightweight virtualization network deployments wherever possible.

This page contains all scripts, resources, and information needed to reproduce or further evaluate the data from the Paper "CControl groups added Latency in NFVs: An update needed?".

All scripts and raw data (for submission only a subset) can be found in the following repository: https://github.com/wiednerf/cgroups-nfv/tree/main
